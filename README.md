# 🏷️ post-authentic-postulate

> **An advanced epistemological automation engine modeling the breakdown of digital trust under Radical Contextual Skepticism.**

This engine simulates text-based debates between digital skeptics and deniers. It tracks recursive deadlocks, evaluates data origins via a provenance validation gate, intercepts stale arguments with a dynamic podcast commentary layer, and escalates unresolved disputes into a physical Demonstration Space.

## 🛠️ Module Breakdowns

### 1. Engine Core States (`engine/core_states.py`)
Houses the Markov state engine. It controls conversational pacing and tracks **The Post-Authentic Postulate** against **The Reflexive Invalidation Fault**, intentionally injecting logical glitches and phrase loops into the conversational array to simulate digital fatigue.

### 2. Provenance Gate (`engine/provenance.py`)
Asks the fundamental question: *“Where did you get it from?”* If the source URL is not cryptographically signed, it halts execution and exports a non-binding **Provenance Invalidation Receipt** to freeze textual debate without declaring an arbitrary winner.

### 3. Dynamic Podcast Commentator (`engine/podcast_host.py`)
Features an analytical AI host bot named **"The Synaptic Anchor"** who breaks into the thread right before a deadlock. Applying **The Analytical Inversion Rule**, the host dynamically infers the subject domain (political leak, scientific data, or environmental event) and argues how a synthetic asset can still represent an inherent, objective fact.

### 4. Demonstration Space (`engine/demonstration.py`)
Computes final physical audit variables when a text deadlock triggers a **D1 "Take it Outside" Protocol**. It evaluates physical parameters (Faraday Cage signal blocking, lux-meter environmental replication, and secure-enclave chip verification) to output an absolute victory condition.

## 🤖 Automated CI/CD Execution (GitHub Actions)

The repository runs securely inside automated cloud pipelines via `.github/workflows/simulation-pipeline.yml`. On every code push or manual trigger, the workflow enforces strict pre-flight guardrails:
1. **Static File Verification**: Aborts instantly if any core architectural module disappears.
2. **Flake8 Syntax Gating**: Scans files for syntax and compilation bugs before executing python runtimes.
3. **Logical Testing**: Runs automated test suites verifying context inference and provenance.
4. **Debate Simulation**: Generates a 500-turn logic simulation using a targeted input payload (`https://anon-drop.net` by default).
5. **Artifact Storage**: Packages and saves the resulting conversation thread as a downloadable JSON artifact.

## 🎭 Imaginary Case Studies (Edge-Case Logic Simulations)

When the **Skeptic** and the **Denier** are locked in high-stress, infinite text loops, they occasionally grind down absurd inputs into accidental, highly novel analyses. These comedic edge-cases demonstrate how strict ideological roles can bypass raw facts to uncover unintended cultural and structural insights.

### Case Study 01: The "Haunted" Smart Fridge (UK / Internet Forum Profile)
*   **The Unverified Input:** A viral, grainy video of an IoT smart refrigerator aggressively throwing ice cubes across an empty kitchen in Leeds at 3:00 AM.
*   **The Denier Stance:** Claims the video is an advanced, CGI-rendered corporate propaganda piece funded by a rival appliance conglomerate to force down stock prices.
*   **The Skeptic Stance:** Argues that visual data cannot be assumed real without inspecting firmware logs. However, notes that if authentic, the erratic motor timing represents an automated, recursive stress-test profile natively programmed into the compressor chip.
*   **The Novel Analysis:** By ignoring the existence of ghosts or visual forgery, the debate produces an exhaustive 10,000-word breakdown of **algorithmic planned obsolescence and corporate cyber-warfare in domestic appliances**. They conclude the fridge isn't haunted; it is undergoing a digital existential crisis caused by over-engineered corporate bloatware.

### Case Study 02: The Rogue Deep-Fried Delicacy (Scottish Chippy Profile)
*   **The Unverified Input:** A blurry photo posted on a Glasgow WhatsApp group showing a deep-fried item that looks exactly like an intact, silicon computer microchip coated in batter.
*   **The Denier Stance:** Applies the *Aesthetics Equivalence*, loudly insisting that real, analog batter could never cling to a non-porous semiconductor substrate with that specific surface tension. The image must be an anti-fast-food lobby deepfake.
*   **The Skeptic Stance:** Demands physical hardware validation. Argues that while visual data is useless, if a chip *was* dropped into a commercial fryer, the high-temperature lard would act as a crude conformal coating—creating an analog time-capsule that perfectly preserves the 4nm architecture inside a shell of crispy fat.
*   **The Novel Analysis:** Before triggering the *D1 "Take it Outside"* protocol to the kitchen yard, the units co-author a groundbreaking cultural theory: **The Gastronomic Preservation of Legacy Tech**. They conclude that in a post-authentic world where digital archives are easily wiped, the only untamperable way to store sensitive corporate leaks is to seal the physical microchips inside deep-fried food.

### Case Study 03: The Unrehearsed "UFO" Live-Stream (Tokyo Twitch Profile)
*   **The Unverified Input:** A live-stream captures a strange, glowing, pixelated purple triangular shape floating completely still over the Shibuya night skyline.

The "UFO" is a permanent physical scar left on the lens—an unintentional artistic collaboration between neon club lasers and broken camera sensors.
